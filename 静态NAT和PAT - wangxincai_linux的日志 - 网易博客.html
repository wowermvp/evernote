静态NAT和PAT  

 
1.       利用静态NAT，是内部服务器SERVER1发布到公网，是外部能够访问到内网的服务器。
2.       使用PAT技术，使PC1和PC2使用同一公网IP能够访问外网。
需求分析：
1.       首先实现局域网内部的通信。、
2.       配置接口的IP地址，做静态NAT，默认路由。

2.        配置接口的IP地址，做静态NAT,，添加默认路由
（R1）   
Router(config)#ip rou   //因为R1的默认路由指向了外网，所以需要配置内网的静态路由
Router(config)#ip route 192.168.2.0 255.255.255.0 192.168.100.1
Router(config)#ip route 192.168.3.0 255.255.255.0 192.168.100.1
Router(config)#ip route 192.168.4.0 255.255.255.0 192.168.100.1
Router(config)#ip nat inside source static 192.168.2.2 202.102.1.3
Router(config)#in               //一个私网IP转换成一个公网IP（静态）
Router(config)#interface f0/0     //连接局域网的接口
Router(config-if)#ip na
Router(config-if)#ip nat i
Router(config-if)#ip nat inside  //在内部接口启用转发
Router(config-if)#exi
Router(config-if)#exit
Router(config)#in
Router(config)#interface f0/1     //连接外网的接口
Router(config-if)#ip na
Router(config-if)#ip nat outside   //在外部接口启用转发
Router(config-if)#
Router(config)#ex
Router(config)#exit
Router#

3.在R1做PAT，使PC1和PC2使用同一IP访问外网
（R1）
Router(config-if)#exit
Router(config)#access-list 1 pe
Router(config)#access-list 1 permit 192.168.4.0 0.0.0.255
Router(config)#access-list 1 permit 192.168.3.0 0.0.0.255
Router(config)#ip nat inside source list 1 in
Router(config)#ip nat inside source list 1 interface f0/1 o
Router(config)#ip nat inside source list 1 interface f0/1 overload
Router(config)#exit
