权限管理命令

命令名称：chmod
英文原意：change the permissions mode of a file
命令所在路径：/bin/chmod
执行权限：所有用户
功能描述：改变文件或目录权限
语法：chmod [{ugoa}{+ - =}{rwx}] [文件或目录]
            [mode=421] [文件或目录]
            -R 递归修改
     权限数字表示
     r------4    w------2      x-----1
范例：$ chmod 755 /tmp/wower             #将/tmp/wower的权限改写成rwxr-xr-x。
          $ chmod -R 777 /tmp/wower        #将/tmp/wower目录及目录下的文件的权限改为rwxrwxrwx。
文件目录权限总结：
           
代表字符权限对文件含义对目录的含义
r读可以查看文件内容可以列出目录中的内容
w写可以修改文件内容可以在目录中创建、删除文件
x执行可以执行文件可以进入目录

命令名称：chown
英文原意：change file ownership
命令所在路径：/bin/chown
执行权限：root
功能描述：改变文件或目录的所有者
语法：chown [用户] [文件或目录]
范例：# chown wower test                     #改变文件test的所有者为wower。
          #chown wower:wower test          #改变文件test的所有者和所属组。

命令名称：chgrp
英文原意：change file group ownership
命令所在路径：/bin/chgrp
执行权限：root
功能描述：改变文件或目录的所属组
语法：chgrp [用户组] [文件或目录]
范例：# chgrp wower test                      #改变文件test的所属组为wower。

命令名称：umask
英文原意：the user file-creation mask
命令所在路径：shell内置命令
执行权限：所有用户
功能描述：显示、设置文件的缺省权限
语法：umask [-S]
            -S  以rwx形式显示新建文件缺省权限
范例：$ umask -S                             #显示新建文件的缺省权限
umask是从权限中“拿走”相应的位,且文件创建时不能赋予执行权限
永久改变umask权限需要在/etc/profile中编辑
例：umask为002(--------w-)与文件权限666(-rw-rw-rw-)得664(-rw-rw-r--),所以umask为002时文件默认权限为664(-rw-rw-r--)。
