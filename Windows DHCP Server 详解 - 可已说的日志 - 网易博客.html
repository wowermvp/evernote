需要使用的命令：    
    1、释放IP地址：Ipconfig /Release
    2、更新IP地址租约：Ipconfig /Rnew
    3、查看本机IP地址配置信息：Ipconfig  /all
什么是DHCP服务器？
答：DHCP（Dynamic Host Configure Protocol，动态主机配置协议）是用于向网络中的计算机分配IP地址及一些TCP/IP参数配置信息。
为什么要使用DHCP服务器？
答：第一、可以大大节省网络管理员的工作量。（包括：大型企业多客户端的情况、局域网多移动设备的情况、局域网跨度大的情况等）
       第二、大幅度提高IP地址的利用率。
       第三、避免了手工配置造成的IP地址冲突和其他错误。
 DHCP服务器工作原理详解
          客户机从DHCP服务器获得IP地址的过程称为DHCP的租约过程。过程分为四个步骤，分别为：①客户机请求IP地址②服务器响应请求③客户机选择IP地址④服务器确定租约。具体如下图：
 

1、客户机请求IP地址。
           DHCP客户机在网络中广播DHCP Discover包请求IP地址。
2、服务器响应。
          服务器收到DHCP Discover包后响应并发送DHCP Offer包。
3、客户机选择IP。
          客户机收到DHCP Offer包后选择IP地址，并发送DHCP Request包。
4、服务器确认租约。
          服务器收到DHCP Request包后发送DHCP Ack/nack包。（确认发送DHCP Ack包、拒绝发送DHCP Nack包）
数据包解释：
          DHCP Discover包：源地址为0.0.0.0 目标地址为255.255.255.255 包含本机MAC地址和计算机名。本包以广播形式发送。
          DHCP Offer包：包含DHCP客户机的MAC地址，用来正确表示客户机；DHCP服务器提供的地址；子网掩码；租约期限；服务器标识符（DHCP服务器的IP地址）；其他可选参数（如网关、DNS服务器）。本包采用广播形式发送。
          DHCP Request包：包含为该客户机提供IP配置服务器的标识符（DHCP服务器的IP地址），服务器在收到包后会检查服务器标识符字段，确定自己提供的IP地址已经被客户机选中。如果选中则保留该IP，如果那些DHCP Offer包被拒绝则取消提供保留IP地址，以用于其他客户端请求。由于此时客户机还没有使用服务器提供的地址，因此这个包还是采用广播形式发送。
          DHCP Ack/Nack包：包含IP地址的有效租约期和其他可能配置信息。当服务器收到DHCP Request包后确认有效便会发送DHCP Ack包进行确认，客户机收到确认信息后便会将从服务器上获得的TCP/IP参数配置上进行使用。如果确认无效则发送DHCP Nack包取消该次IP地址请求，客户机收到拒绝包后便会重新从DHCP Discover开始重新申请。由于此时客户机还没有使用服务器提供的地址，所以这个包还是采用广播形式发送。         
IP地址租约的更新机制：
     当客户重启或者租期达到50%的时候，就会重新以单播的方式向服务器更新租约，客户机直接向提供租约的服务器发送DHCP Request包，要求更新现有的租约。这个时候会有两种现象出现：①如果服务器收到租约信息，它便会发送DHCP确认信息给客户机更新客户机租约。②如果客户机无法与提供租约的服务器取得联系，则客户机继续使用该IP地址知道租期到达87.5%时，它便会向网络上所有的DHCP服务器广播的方式发送DHCP Request包，已更新现有的地址租约。这个时候又会出现两种问题：①如果有服务器响应客户机的DHCP Request包请求，那客户机便会使用该服务器提供的地址信息更新现在的租约。②如果没有其他服务器响应并租约到期，客户机便会放弃正在使用的IP地址，开始新的租约过程。

IP地址租约的释放：
       如果在租约期想要释放该IP地址，可在客户机上执行“ipconfig /release"命令。使客户机向DHCP服务器发送DHCP Release包，并释放其租约。
       如果客户机在租约时间内关闭（并没有不更新租约），那么在租约期满后，DHCP服务器可能将客户机使用的IP地址分配给其他客户机。如果客户机不发送DHCP Release包，那么它在重启时，仍可尝试继续使用上一次使用过的地址。
