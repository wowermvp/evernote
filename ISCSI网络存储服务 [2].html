ISCSI网络存储服务

传统的存储成为DAS（direct attach storage，直接附加存储），也就是硬盘，通过与计算机主板的I/O接口（如IDE、SCSI）相连，由本机操作系统负责读写管理。

现在企业中数据信息不断增长，大量的数据需要备份、恢复。以及面临着扩容，灾难恢复、跨平台使用的需求时，网络存储技术应运而生，最常见的网络存储是NAS和SAN

 

SAN（Storage Area Network，存储区域网络）

多采用高速光纤通道，对速率、冗余性要求高

使用iSCSI存储协议，块级传输

在客户端进行文件系统管理


使用Linux系统构建ISCSI服务器时要先构建ISCSI对象（Target），然后客户机分割逻辑卷存储空间LUN（logical unit逻辑存储单元）并设置权限。挂靠在ISCSI对象中的每个LUN建议只分配给一台客户机使用

 

NAS（Network Attachment Storage，网络附加存储）

采用普通以太网，对速率、冗余无特别要求

使用NFS、CIFS共享协议，文件级传输

在服务端进行文件系统管理

CIFS是微软的文件共享协议；NFS协议最初由SUN公司开发，基于RPC通道将远程主机的文件夹映射为本地目录，绝大多数Linux和UNIX系统默认都支持NFS协议，无论是CIFS和NFS，都能够跨平台使用，每个共享文件夹能够被多个客户机同时使用。

 

本章讲如何构建iscsi服务器以及客户端如何使用

 

构建ISCSI服务器

 

1、准备作为LUN发布的存储设备

可以是硬盘、分区、逻辑卷、RAID阵列……

使用mdadm工具可创建软RAID阵列设备

-C，--create 创建新的RAID设备

-n，--raid-devices= 阵列所使用的磁盘数量

-l，--level= RAID级别

-Q，--query 查询设备的RAID信息

-D，--detail 查询RAID设备的详细信息

-S，--stop 解散、停止/删除RAID阵列



 

2、创建iSCSI对象target （tgtd端口号：3260）

软件包：scsi-target-utils   yum -y install scsi-target-utils

使用tgtdm工具创建target

-L、--lld 指定驱动类型，如“-L iscsi”

-o、--op 指定操作类型，如“-o new”

-m、--mode 指定管理目标，如“-m target”

-t、--tid 指定对象ID号，如“-t 1”

-T、--targetname 指定iSCSI对象的名称



 

3、为客户机分配LUN

仍然使用tgtdam管理工具

-l、--lun 指定LUN的ID号，如“-l 1”

-b、--backing-store 指定块设备位置，如“-b /dev/md0”

-I、--initiator-address 指定客户机地址，如“-I 192.168.4.22”

-m logicalunit 可简写为-m lu，表示管理目标位LUN

-o bind 用来绑定客户机地址

完成以上操作，再次查看target信息，会发现新分配的LUN1的信息，对应的LUN类型为disk

4、保存target和LUN设置

配置文件 /etc/tgt/targets.conf

确保tgtd服务能自动运行

 

使用iSCSI存储设备

1、发现并连接iSCSI对象

软件包：iscsi-initiator-utils, lsscsi

使用iscsidm工具查找、连接target

[root@localhost ~]# service iscsi restart 启动iscsi服务。

 

[root@localhost ~]# iscsiadm -m discovery -t sendtargets -p 192.168.4.11

192.168.4.11:3260,1 iqn.2011-10.com.benet.store:lvm

其中选项“-m discover”表示发现查找

“-t sendtargets”表示发布的target（可缩写为-t st）

“-p”用来指定iscsi服务器地址

结果列出了iscsi服务器发布的，本机可以使用的iscsi对象名称，下一步就是根据此名称连接到指定的iscsi存储对象，使用-m node 选项便是管理目标位节点，选项“-l”或者“--login”表示连接/登录

[root@localhost ~]# iscsiadm -m node -T iqn.2011-10.com.benet.store:lvm --login

Logging in to [iface: default, target: iqn.2011-10.com.benet.store:lvm, portal: 192.168.4.11,3260]

Login to [iface: default, target: iqn.2011-10.com.benet.store:lvm, portal: 192.168.4.11,3260] successful.

 

确认已连接的iSCSI设备

使用lsscsi确认带 IET 标记的设备

[root@localhost ~]# lsscsi

[1:0:0:0]    cd/dvd  NECVMWar VMware IDE CDR10 1.00  /dev/sr0

[2:0:0:0]    disk    VMware,  VMware Virtual S 1.0   /dev/sda

[2:0:1:0]    disk    VMware,  VMware Virtual S 1.0   /dev/sdb

[4:0:0:0]    storage IET      Controller       0001  -

[4:0:0:1]    disk    IET      VIRTUAL-DISK     0001  /dev/sdc

查看自动建立的配置文件

[root@localhost ~]# ls -R /var/lib/iscsi/nodes/

/var/lib/iscsi/nodes/:

iqn.2011-10.com.benet.store:lvm

/var/lib/iscsi/nodes/iqn.2011-10.com.benet.store:lvm:

192.168.4.11,3260,1

/var/lib/iscsi/nodes/iqn.2011-10.com.benet.store:lvm/192.168.4.11,3260,1:

default

 

2、使用iSCSI存储设备

格式化及挂载访问

fdisk、mkfs、mount操作，与普通设备无异

若iSCSI设备已包含文件系统，则无需再格式化分区,

[root@localhost ~]# fdisk -l /dev/sdc

……

   Device Boot      Start         End      Blocks   Id  System

/dev/sdc1               1        1017     4193060   83  Linux

 

格式化

[root@localhost ~]# mkfs -t ext3 /dev/sdc1

 

创建目录挂载文件系统

[root@localhost ~]# mkdir /opt/data

[root@localhost ~]# mount /dev/sdc1 /opt/data

[root@localhost ~]# cat /etc/mtab

/dev/sdc1 /opt/data ext3 rw 0 0

 

3、保存iscsi的连接配置

为了是iscsi存储设备使用正常，客户机应确保iscsi服务能够自动启动，并将挂载iscsi设备中的文件系统的操作写入/etc/fstab文件。由于iscsi设备的特殊性，fstab文件中的配置记录应启用“_netdev”挂载参数，表示这是一个网络设备

[root@localhost ~]# vi /etc/fstab

……

/dev/sdc1         /opt/data        ext3    defaults,_netdev   0 0

    可能客户端新添加硬盘后，网络存储设备不再是/dev/sdc1，这时候使用设备的UUID号更加可靠

[root@localhost ~]# blkid /dev/sdc1

/dev/sdc1: UUID="93846775-458c-405d-bd46-2de7712dfeb6" TYPE="ext3"

[root@localhost ~]# vi /etc/fstab

……

UUID=93846775-458c-405d-bd46-2de7712dfeb6   /opt/data   ext3  defaults,_netdev  0 0

 

第一次重启之前，建议先断开iSCSI，避免假死现象

[root@localhost ~]# umount /dev/sdc1

[root@localhost ~]# iscsiadm -m node -T iqn.2011-10.com.benet.sto

re:lvm  --logout

 

[root@localhost ~]# chkconfig --level 35 iscsi on

[root@localhost ~]# reboot

 

注意：如果有两个客户端使用同一个iscsi网络存储设备，其中一个客户机在这个存储设备中创建了新文件，另一个客户机需要重新挂载才能看见新创建的文件
